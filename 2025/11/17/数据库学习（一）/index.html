数据库相关概念

数据库基础

数据库

数据库(Database，DB)是按照数据结构来组织、存储和管理数据的建立在计算机存储设备上的仓库。是长期储存再计算机内、有组织的、可共享的大量的数据集合。

数据管理系统

为了管理我们的数据库，就引入了数据管理系统

数据管理系统（database management system，DBMS）:位于用户和操作系统之间的一层数据管理软件。其和操作系统一样都属于计算机基础软件，也是大型复杂的软件的系统。

用于创建、管理和维护数据库的软件系统，它能有效组织数据、控制数据访问，并保障数据的安全性与完整性。

例如：我们连接到一个DBMS，然后下达查询或修改数据的指令，DBMS执行指令并返回结果，

主要功能：

  1. 数据定义（ddl语言定义）
  2. 数据库操作（增删改查）
  3. 数据库组织、存储和管理
  4. 数据库的运行管理与控制（保证数据安全性）
  5. 数据库建立与维护

数据库管理系统又分为两类：关系型和非关系型（NoSQL）

类型 代表产品 核心特点 适用场景
关系型数据库 MySQL、Oracle、PostgreSQL 数据以 “表格” 形式存储，表间通过关联关系（如主键、外键）连接，支持 SQL 语言 需严格数据结构的场景，如电商订单、银行交易、企业 ERP
非关系型数据库（NoSQL） MongoDB（文档型）、Redis（键值型）、Neo4j（图型） 不依赖传统表格结构，存储形式灵活（如文档、键值对、图形） 大数据量、高并发或非结构化数据场景，如社交软件消息、实时缓存、推荐系统

数据库系统

数据库系统（DBS）是一个由数据库、数据库管理系统（DBMS）、应用程序、用户和硬件组成的完整数据处理体系，核心是通过 DBMS 高效管理和利用数据库中的数据。

特点：

  1. 数据的结构化：采用一定的数据模型

    数据的整体结构化是数据库的主要特征之一，它不再仅仅针对某一个应用，而是面向全组织，不仅数据内部结构化，整体也是结构化的，数据之间具有联系，数据记录可以变长，数据的最小存取单位是数据项。

  2. 数据的共享性高、冗余度低、易扩展

    数据面向整个系统，可以被多个用户、多个应用共享使用。

    数据共享的好处：

    • 减少数据冗余，节约存储空间。

    • 避免数据之间的不相容性与不一致性。

    • 使系统易于扩充。

  3. 数据的独立性高：包括物理的独立性和逻辑独立性

    • 物理独立性：用户的应用程序与存储在磁盘上的数据库中的数据是相互独立的
    • 逻辑多理性：用户的应用程序与数据库的逻辑结构是相互独立的

  4. 数据由DBMS统一管理和控制

    数据库管理系统提供的数据控制功能。

    • 数据的安全性（Security）保护：保护数据以防止不合法的使用造成的数据的泄密和破坏。

    • 数据的完整性（Integrity）检查：保证数据的正确性、有效性和相容性。

    • 并发（Concurrency）控制：对多用户的并发操作加以控制和协调，防止相互干扰而得到错误的结果。

    • 数据库恢复（Recovery）技术：将数据库从错误状态恢复到某一已知的正确状态。

  5. 数据的最小存取单位是数据项

数据表

关系数据库没有数据表，关键字、主键、索引等也就无从谈起，数据表是关系数据库中一个非常重要的对象，是其它对象的基础，也是一系列二维数组的集合，用来存储、操作数据的逻辑结构。根据信息的分类情况。一个数据库中可能包含若干个数据表，每张表是由行和列组成，记录一条数据数据表就增加一行，每一列是由字段名和字段数据集合组成，列被称之为字段，每一列还有自己的多个属性，例如是否允许为空、默认值、长度、类型、存储编码、注释等.

数据库访问技术

“数据库访问技术（Database Access Technology）”是指应用程序与数据库之间进行数据交互的技术和方法，它负责在应用程序与数据库管理系统（DBMS）之间建立联系、执行SQL语句、传递数据结果。

简单来说，它就是“桥梁”：

应用程序（如Java、Python、C#） ↔ 数据库（如MySQL、Oracle、SQL Server）

通过数据库访问技术，程序可以实现：

  • 连接数据库（Connection）
  • 发送 SQL 语句（Statement）
  • 获取结果集（ResultSet）
  • 对数据进行增删改查操作（CRUD）

数据库–视图

数据库–视图的基本概念以及作用

视图是一个虚拟的表（virtual table），它不直接存储数据，而是基于一条或多条 SELECT 查询语句 动态生成的数据集。

可以像操作表一样操作视图（如 SELECT、UPDATE），但它本质上只是一个保存了查询语句的逻辑结构。

视图（子查询）：是从一个或多个表导出的虚拟的表，其内容由查询定义。具有普通表的结构，但是不实现数据存储。
对视图的修改：单表视图一般用于查询和修改，会改变基本表的数据，
多表视图一般用于查询，不会改变基本表的数据。

SQL语言

介绍

SQL（Structured Query Language，结构化查询语言） 是一种用于 管理、查询和操作关系型数据库 的标准语言。
它由 IBM 在 1970 年代开发，后被 ANSI（美国国家标准协会）和 ISO（国际标准化组织）采纳为数据库标准语言。

SQL 被几乎所有关系型数据库系统支持，如：

  • MySQL
  • Oracle
  • Microsoft SQL Server
  • PostgreSQL
  • SQLite

尽管不同系统之间可能存在语法差异，但核心语法几乎一致。

SQL语言特点

  1. 综合统一
  2. 高度非过程化
  3. 面向集合的操作方式
  4. 以同一种语法结构提供两种使用方式，SQL语言既是自含式语言，又是嵌入式语言
  5. 语言简捷，易学易用

主要语言分类

SQL语言功能非常丰富，通常分为以下四类：

类别 全称 功能说明 示例
DDL Data Definition Language 数据定义语言：定义数据库结构 CREATE TABLE、ALTER TABLE、DROP TABLE
DML Data Manipulation Language 数据操作语言：插入、更新、删除数据 INSERT、UPDATE、DELETE
DQL Data Query Language 数据查询语言：查询数据 SELECT
DCL Data Control Language 数据控制语言：设置权限、管理事务 GRANT、REVOKE、COMMIT、ROLLBACK

注意：

SQL语句一般由关键字、标识符、运算符、表达式等组成，语句以 分号（;） 结束。

SQL 不区分大小写，但为了规范化，关键字通常写为大写。

定义语言(DDL)

MySQL数据库学习笔记（三）——数据定义语言DDL（持续更新）

介绍

DDL（Data Definition Language，数据定义语言） 是用于定义和管理数据库对象（如表、视图、索引等）的语言。

一个关系数据库管理系统的实例（Instance）中可以建立多个数据库，一个数据库中可以建立多个模式，一个模式下通常包括多个表、视图和索引等数据库对象。因此SQL的数据定义功能包括定义模式、定义表、定义视图和定义索引。视图是基于基本表的虚表，索引是依附于基本表的，因此SQL通常不提供修改视图定义和修改索引定义的操作。用户如果想修改视图定义或索引定义，只能先将它们删除掉，然后再重建。不过有些关系数据库产品Oracle允许直接修改视图定义。

数据定义（DDL）

定义

数据定义语言（DDL）是SQL语言的一个重要组成部分，用于定义和管理数据库对象，包括表（表结构和约束）、视图、索引、存储过程、函数等。SQL Server的DDL语句主要由以下几个部分组成：

  1. CREATE：用于创建数据库对象。
  2. ALTER：用于修改数据库对象。
  3. DROP：用于删除数据库对象。

在SQL Server中，DDL操作是由系统目录和系统存储过程来执行的。DDL语句不会自动提交事务，需要手动执行COMMIT语句来提交事务。

  • IX_Employee_Name：索引的名称。
  • [dbo].[Employee]：指定索引所属的表。
  • [Name]：指定要创建索引的列

GO

它不是 SQL 语句本身的一部分，而是 SQL Server Management Studio (SSMS) 或 sqlcmd 工具 用来分隔批处理（Batch） 的指令（命令终止符）

作用：告诉 SQL Server 工具：上面的一段 SQL 语句到这里结束了，请立即执行。

视图

视图是一个虚拟表，其内容由查询定义。但视图并不存储数据，视图数据来自由定义视图的查询所引用的表。关系数据库管理系统执行CREATE VIEW语句时只是把视图定义存入数据字典，并不执行其中的SELECT语句。在对视图查询时，按视图的定义从基本表中将数据查出。视图能够简化用户的操作，使用户能以多种角度看待同一数据，视图对重构数据库提供了一定程度的逻辑独立性，能够对机密数据提供安全保护，适当的利用视图可以更清晰的表达查询。

  • CREATE VIEW：创建视图的语句。

  • [dbo].[EmployeeView]：指定视图的名称。

  • SELECT ID, Name, Age, Salary：视图的查询语句。

  • FROM [dbo].[Employee]：指定查询的数据源。

  • WHERE Salary > 5000：指定查询的条件。

语句部分 解释
CREATE VIEW View_EdsProd 创建一个名为 View_EdsProd 的视图。视图类似一个保存了查询语句的“虚拟表”。
AS 关键字，表示视图的定义部分（即接下来的查询）开始。
SELECT A.* 从表 Tab_A（起别名为 A）中选择全部列。* 代表所有字段。
B.Code AS bcode 从表 Tab_B（起别名为 B）中选择字段 Code，并将它在视图中重命名为 bcode。
B.Name AS bName 同理，从表 Tab_B 中选择字段 Name，并重命名为 bName。
FROM Tab_A A 数据来源表 1：Tab_A，并取别名为 A（方便后面引用）。
LEFT JOIN Tab_B B ON (A.Mid = B.Mid) 将表 Tab_A 与表 Tab_B 按字段 Mid 进行左连接（LEFT JOIN）。左连接的含义是： → 保留左表（A）的所有行； → 如果右表（B）中有匹配的 Mid，则一起显示； → 如果右表没有匹配项，则右表字段为 NULL。
WHERE A.Mid > 1 过滤条件，只保留 A.Mid 值大于 1 的记录。
